## What This Is

SkillWatch is a free, open-source Python CLI tool that monitors the **content at external URLs** referenced by AI skills and MCP tools. It detects when URL content changes and flags suspicious patterns (new download commands, script injection, credential harvesting).

It does NOT scan skill code or metadata — that's what tools like MCP-Scan/Snyk Agent Scan do. SkillWatch monitors what those skills **point to**. The two are complementary.

**Why it exists:** In June 2026, security firm AIR demonstrated that a fake skill could bypass every major scanner (Cisco, NVIDIA, skills.sh) by keeping its code clean and swapping the content at an external URL after distribution. No free tool monitors URL content post-installation. SkillWatch fills this gap.

**Business model:** Not SaaS. Free tool + TIL article → AI governance consulting pipeline. Target: £300/month to send home to parents in SA. See ECONOMICS.md and PMF-ASSESSMENT.md for the full analysis.

## Security Architecture

### SSRF Protection (ssrf.py)

- Blocks 16 IP networks: all RFC 1918, loopback, link-local, multicast, 6to4, NAT64, unspecified
- Unwraps IPv4-mapped IPv6 (e.g. `::ffff:127.0.0.1`)
- Rejects credentials in URLs (`parsed.username`/`parsed.password`)
- Rejects non-standard numeric hostnames (decimal `2130706433`, hex `0x7f000001`)
- Catches `UnicodeError` from malformed hostnames (e.g. `.localhost`)
- Only allows `http://` and `https://` schemes

### DNS Pinning (ssrf.py PinnedDNSAdapter)

- DNS resolved once in `validate_url`, IP pinned in `ValidatedURL` dataclass
- `PinnedDNSAdapter` rewrites URL to use pinned IP, sets `assert_hostname`/`server_hostname` for TLS SNI
- No global state modified (no `socket.getaddrinfo` patching) — thread-safe
- Each redirect hop gets its own fresh session with a new pinned adapter

### Redirect Handling (fetcher.py)

- Manual redirect following (not `allow_redirects=True`)
- Max 5 hops, each destination validated BEFORE the request is made
- `timeout=(connect, read)` tuple prevents infinite streaming DoS

### Content Security

- Escape sequences stripped at two points: fetch time (fetcher.py) and display time (formatter.py)
- Regex covers CSI, OSC, DCS, Fe, C1 families (prevents OSC 52 clipboard write)
- All URLs escape-stripped before terminal display (`_safe()`/`_safe_url()`)
- Raw HTML stored for old-vs-new comparison (detector only flags NEW suspicious elements)
- Snapshot pruning: max 50 per URL (prevents unbounded disk growth)
- SQLite uses `IMMEDIATE` isolation with 30s timeout (prevents concurrent duplicate alerts)

### Input Validation (parser.py)

- `yaml.safe_load()` exclusively (never `yaml.load()`)
- 1MB file size cap on JSON/YAML configs (billion-laughs mitigation)
- `_walk_json` depth limit at 50 (prevents RecursionError)
- `RecursionError` and `MemoryError` caught in JSON/YAML parsing

## Key Technical Decisions

| Decision | Choice | Reason |
|---|---|---|
| Text extraction | trafilatura (bytes passed directly) | Handles encoding detection internally. Strips boilerplate/nav/ads. |
| Hash target | Extracted text, not raw HTML | Raw HTML changes constantly (CSRF tokens). Text is stable. |
| Storage | SQLite, `IMMEDIATE` isolation | Zero infrastructure. Prevents concurrent scan conflicts. |
| DNS pinning | URL rewriting + assert_hostname | Thread-safe. No global monkey-patching. |
| Detection | 6 rule-based patterns, no LLM | Keeps tool free, offline, dependency-light. |
| Scheduling | cron, not daemon | Simpler. Users know cron. |
| Config | CLI flags only | No config file in v0.1. `--ignore-pattern` for noise suppression. |
| Dependencies | 4 direct, all Apache/MIT/BSD | `trafilatura>=2.0,<3`, `requests>=2.32.4` |

### Known limitations (documented, accepted)

- CLI only (no web UI)
- 6 regex-based patterns (no LLM analysis)
- Does not scan skill code/metadata (complementary to MCP-Scan)
- Evasion possible via User-Agent detection
- Zero community adoption (new project, 11 commits)

### Gotchas for the next developer

- `@responses.activate` must be applied per-method, not per-class (pytest doesn't support class-level). This was the cause of 10 silently skipped tests in the first audit.
- The `PinnedDNSAdapter` rewrites URLs to use the pinned IP. Tests using `responses` must register mocks against `https://{MOCK_IP}/path` (see `tests/conftest.py`).
- `store.add_url` returns `(id, is_new)` tuple, not just `id`.
- `get_latest_good_snapshot` skips error snapshots. `get_latest_snapshot` includes them.
- SQLite uses `IMMEDIATE` isolation — writes are serialised. This is intentional.
